# Roadmap

## 원칙
초기에는 자동화보다 흐름 검증을 우선한다. 따라서 `입력 -> 조회 -> 분석 -> AI 리포트` 순서로 구현한다.

## Phase 1. App Shell
- 기본 레이아웃 구성
- 홈 화면을 제품 방향에 맞게 교체
- 공통 스타일, 컬러, 타이포 정리

## Phase 2. Manual Input
- 월 소득 입력 폼
- 고정지출 입력 폼
- 거래내역 수동 입력 폼
- mock 데이터 기반 목록 조회

## Phase 3. Dashboard
- 이번 달 총지출
- 남은 예산
- 고정비 / 변동비 비율
- 카테고리 TOP 3 카드

## Phase 4. CSV Import
- CSV 파일 업로드 UI
- 필드 매핑 규칙
- 업로드 후 거래내역 병합

## Phase 5. Analysis
- 카테고리 집계
- 지난달 대비 증감
- 이상 지출 탐지
- 절감 후보 계산

## Phase 6. AI Report
- 월말 요약 리포트 생성
- 절감 팁 문장화
- 다음 달 예산 제안

## 현재 추천 시작점
1. App Shell
2. 거래내역 수동 입력
3. 대시보드 mock 연결
4. 이후 DB 도입
