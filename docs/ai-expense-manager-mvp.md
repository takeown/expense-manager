# 개인용 AI 지출관리 앱 MVP 기획서

## 1. 프로젝트 개요

자산과 소비 내역이 여러 곳에 흩어져 있는 개인 사용자를 위해,  
월 소득 / 카드 소비 / 계좌이체 / 고정지출을 입력받아  
지출을 분석하고 절감 가능한 항목을 추천하는 **개인용 AI 지출관리 앱**을 만든다.

핵심은 완전한 금융 API 자동연동이 아니라,  
**수동 입력 + 파일 업로드 + AI 분석**으로 빠르게 실사용 가능한 MVP를 만드는 것이다.

---

## 2. 목표

### 핵심 목표
- 이번 달 돈이 어디서 많이 나갔는지 한눈에 파악
- 고정비 / 변동비를 구분
- 줄일 수 있는 지출 항목 추천
- 월말에 AI가 요약 리포트 제공

### 제외 범위
초기 MVP에서는 아래는 하지 않는다.
- 정식 마이데이터 API 연동
- 오픈뱅킹 직접 연동
- 실시간 카드 승인 연동
- 투자 포트폴리오 자동 수집
- 가족 공유/멀티 유저 기능

---

## 3. 대상 사용자

### 1차 타겟
- 본인 혼자 쓰는 개인용 서비스
- 자산이 여러 계좌/증권/연금에 흩어져 있음
- 매달 소비가 어디서 새는지 알고 싶음
- 절약 가이드를 받고 싶음

### 사용자 문제
- 카드, 계좌, 현금 지출이 분산되어 관리가 어려움
- 고정비가 얼마나 나가는지 감이 잘 안 옴
- 과소비 원인을 명확히 모르겠음
- 자산 전체를 한 번에 모아보긴 어려우나, 소비 분석은 필요함

---

## 4. 핵심 가치 제안

이 앱은 “계좌를 다 모아서 보여주는 것”보다  
**“돈이 어디서 새는지 파악하고 줄이는 행동을 유도하는 것”**에 집중한다.

예시:
- “이번 달 배달비가 최근 3개월 평균보다 11만 원 높아요.”
- “구독료 3건 중 1건은 최근 사용 흔적이 적어 보여요.”
- “고정비 비중이 월 소득 대비 52%로 높아요.”
- “이번 달 병원비는 일회성 지출 가능성이 높아 과소비로 보지 않아도 됩니다.”

## 5. MVP 범위

## 5-1. 입력 기능
- 월 실수령 소득 입력
- 고정지출 입력
  - 월세
  - 관리비
  - 보험
  - 통신비
  - 구독료
  - 대출 상환액
- 거래내역 입력
  - 카드 소비내역 수동 입력
  - 계좌이체 내역 수동 입력
  - 현금지출 수동 입력
- 거래내역 파일 업로드
  - CSV 우선 지원
  - 추후 엑셀 지원 가능

## 5-2. 분석 기능
- 거래내역 카테고리 자동 분류
- 고정비 / 변동비 자동 분리
- 월 총지출 계산
- 카테고리별 지출 집계
- 지난달 대비 증감 분석
- 절감 후보 항목 추천
- AI 월말 요약 리포트 생성

## 5-3. 조회 기능
- 이번 달 요약 대시보드
- 거래내역 리스트
- 카테고리별 지출 현황
- 자산 스냅샷 조회
- 월별 리포트 조회

---

## 6. 비기능 요구사항

- 개인용 기준으로 빠르게 입력 가능해야 함
- 모바일/데스크탑 둘 다 보기 쉬워야 함
- 거래내역 원본은 최소한만 저장
- 민감정보는 마스킹하거나 저장 최소화
- AI 결과는 “조언” 성격이며 최종 의사결정은 사용자 본인

## 7. 화면 설계

## 7-1. 온보딩 / 초기 설정
목적:
- 첫 사용 시 기본 정보 입력

입력 항목:
- 월 실수령 소득
- 월 고정지출
- 자주 쓰는 카테고리 규칙(선택)

## 7-2. 대시보드
표시 항목:
- 이번 달 총지출
- 이번 달 남은 예산
- 고정비 / 변동비 비율
- 가장 많이 쓴 카테고리 TOP 3
- 절감 추천 카드
- AI 한줄 총평

예시 문구:
- “이번 달은 식비와 쇼핑이 증가했습니다.”
- “고정비 비중이 높아 통신/구독 점검을 추천합니다.”

## 7-3. 거래내역 입력/업로드
기능:
- 직접 추가
- CSV 업로드
- 최근 입력 목록 확인
- 자동 카테고리 추천
- 규칙 저장
  - 예: 쿠팡 → 쇼핑
  - 예: 배달의민족 → 식비
  - 예: SKT → 통신비

## 7-4. 지출 분석
표시 항목:
- 카테고리별 지출
- 지난달 대비 변화
- 변동비 이상치 탐지
- 반복 구독 탐지
- 절감 가능 금액 추정

## 7-5. 자산 스냅샷
입력 항목:
- 현금
- 예금
- 주식
- 연금/IRP
- 청약
- 대출 잔액

특징:
- 실시간 연동 없음
- 월 1회 수동 업데이트 기준

## 7-6. 월말 리포트
포함 내용:
- 이번 달 소비 요약
- 증가 원인
- 줄일 수 있는 항목
- 다음 달 예산 제안
- AI 코멘트

## 15. Codex 프롬프트 예시

### 프롬프트 1
이 프로젝트는 개인용 AI 지출관리 앱 MVP다.  
Next.js + TypeScript + Tailwind + Prisma + PostgreSQL 기준으로 시작한다.  
먼저 전체 폴더 구조와 초기 라우팅 구조를 잡아줘.  
필요한 페이지는 아래와 같다.
- 대시보드
- 거래내역 입력
- 거래내역 목록
- 자산 스냅샷
- 월말 리포트

### 프롬프트 2
Prisma schema를 작성해줘.  
필요한 모델은 아래와 같다.
- User
- MonthlyIncome
- FixedExpense
- Transaction
- Category
- CategoryRule
- AssetSnapshot
- MonthlyReport

관계와 인덱스를 적절히 설계하고, 개인용 MVP 수준으로 너무 복잡하지 않게 작성해줘.

### 프롬프트 3
거래내역 CSV 업로드 기능을 구현해줘.  
요구사항:
- CSV 업로드
- 헤더 기반 컬럼 매핑
- 날짜/금액/가맹점명 필수
- 미리보기 후 저장
- 에러 행 표시

### 프롬프트 4
대시보드 화면을 만들어줘.  
보여줘야 할 항목:
- 이번 달 총지출
- 이번 달 남은 예산
- 고정비 / 변동비
- 카테고리별 상위 5개
- 절감 추천 카드 3개

### 프롬프트 5
OpenAI API를 연동해 거래내역을 카테고리 분류하는 서버 함수를 만들어줘.  
입력은 merchant_name, description, amount 이고  
출력은 category, confidence, reason 형태의 구조화된 JSON 이어야 한다.

## 18. 한 줄 요약

이 프로젝트는  
**“자동 금융 연동 없이도, 수동 입력 + CSV 업로드 + AI 분석으로 충분히 실용적인 개인용 지출관리 앱을 만든다”**  
를 목표로 한다.
